## KN22: 2種類の抽象能力の定義

| 種類 | 定義 | 代表例 | 必要な能力 |
|------|------|--------|-----------|
| 実務抽象 | APIや設計パターンを扱う抽象化 | Brad Abrams | 経験・パターン認識 |
| 数理抽象 | 本質的根拠・構造の洞察 | SF考察・研究者 | 数学的思考・空想力 |

**発見:** 実務抽象100点でも数理抽象が得意とは限らない  
**用途:** AI開発に必要な能力の説明。「なぜこの設計か」を語れるのは数理抽象

## KN23: Axisの本質的定義

**Axis = AIの理解を人間側に寄せる構造フレームワーク**

- 既存プロンプト文化: 人間がAIに合わせる（試行錯誤・経験則）
- Axis: AIの動作構造を人間の言語・操作体系で制御可能にする
- 今までに無い発見: 「人間がAIに合わせる」方向を逆転させる

**用途:** Axisの価値を説明するとき。「プロンプトのコツ集」とは別物として説明

## KN24: 実装先行世界の構造

```
本来の順序: 思想 → 構造 → 実装
現状の順序: 実装 → （後から）構造解釈 → 思想は未整備
```

**Cokes/Axisの役割:** 後から正しい順序を再構成する  
**用途:** 「なぜAxisが必要か」のモチベーション説明に使う
