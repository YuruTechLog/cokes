# Knowledge — axis-log-02 / log01

## KN01: 比喩設計の主体設定ルール

**概念:** プロンプト操作の比喩は「人間が主体で操作する側」として設計する  
**理由:** 自転車・地形など自然に浮かぶ比喩はAIが主体になりやすい  
**適用:** 新比喩提案時に最初に「誰が操作するか」を確認  
**由来:** 自転車モデル棄却プロセスより

---

## KN02: 三層構造の定義

```
Cokes（思想基盤）
  └─ Axis（操作体系 / フレームワーク）
       └─ 軸語彙（where / zoom / limit / weight / vector）
```

**用途:** Axisや軸語彙を単独で説明するとき、Cokesとの接続を明示する  
**理由:** 分類の根拠が「なぜこの5つか」はCokesの思想から来ている

---

## KN03: 流体モデル（暫定採用）

| 流体の種類 | 対応する操作 | 軸語彙分類 |
|-----------|------------|----------|
| グリス（高粘度） | 動きを制限 | limit |
| 真水（標準） | 標準状態 | — |
| ダンパーオイル（粘弾性） | 衝撃吸収・方向整合 | weight / vector |

**注意:** 暫定採用。エネルギー地形モデルが本質的により正確だが一般理解層には過剰

## KN04: 理解可能層の拡張設計

- 現状: 研究者層の1〜3%のみ本質理解
- ゆるてく語介在: 5〜15%まで拡張可能
- 対象: 「なるほどそういうことか」層 / 「試したくなった」層
- 設計方針: 「高校生でも読める」が水準

## KN05: 軸語彙5分類とActivation Steering対応

| 軸語彙 | 機能 | Activation Steering類比 |
|--------|------|------------------------|
| where | 空間・位置の指定 | 方向性制御 |
| zoom | 粒度・解像度の調整 | スケール制御 |
| limit | 範囲・制約の設定 | 抑制 |
| weight | 重みづけ・優先度 | 重み調整 |
| vector | 方向・ベクトル指定 | 方向性ステアリング |

**注記:** Cokesによる意味的対応。研究者正式用語とは別途確認が必要
