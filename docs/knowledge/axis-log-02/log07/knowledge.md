# Knowledge — axis-log-02 / log07

## KN26: Axisの機能定義（研究者暗黙操作の形式化）

**Axis = 研究者が暗黙にやっている「構造からの設計」を、誰でも実行可能な言語・体系にしたもの**

- 研究者: 構造（意味空間の性質）→ プロンプト設計（形式化なし）
- 一般ユーザー: プロンプトの経験則 → 模倣（構造理解なし）
- Axis: 構造を言語化 → 誰でも構造から設計可能に

**用途:** Axisの必要性を説明するとき。研究者と一般ユーザーの「橋渡し」として定義

## KN27: プロンプト集とAxisの関係性

| 比較軸 | プロンプト集 | Axis |
|--------|-----------|------|
| 内容 | 経験則・効くプロンプト | 構造・定理・操作体系 |
| 由来 | 試行錯誤の蓄積 | 研究者暗黙知の形式化 |
| 使い方 | コピーして使う | 構造を理解して設計する |
| 位置づけ | Axisの表層 | 設計図 |

**説明の注意:** 「Axisが上」とは言わない。「役割が違う。入口と設計図」として説明

## KN28: SNSによるプロンプト知識断片化の説明

```
研究者の暗黙操作
  ↓ SNSで断片的に漏れる
「〇〇と書くといい」「このプロンプトが効く」
  ↓ 経験則として蓄積
構造なきプロンプト文化
```

**用途:** 「なぜプロンプト集では限界があるか」の説明  
**Axisの解答:** 断片を構造に接続する = 一つの操作の「なぜ効くか」が分かる

## KN29: 実装先行に対する思想の位置づけ

**「実装が先に来た」= 思想の無効化ではない。思想の裏付け（事後証拠）**

- Anthropicが実装 → 「その構造は正しかった」の証明
- 「ただの実装だ」と感じても、思想・概念設計の価値は独立して存在する
- 実装と概念設計は別次元
