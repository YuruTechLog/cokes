## 横断テーマ

### 1. 「比喩の主体問題」

全ログを通じて: プロンプト操作の比喩は「人間が主体で操作する」設計が必須。
- 棄却例: 自転車（AIが乗る）/ 地形（プロンプトが地形を変える）
- 採用例: 媒質操作（人間が粘度を変える）/ 光照射（人間が光を当てる）
- **ルール:** 比喩設計前に「誰が操作するか」を先に決める

### 2. 「形式化の価値」

全ログを通じて: Cokes/Axisの独自性は「発見」ではなく「形式化」。
- 概念は学術領域に存在済み
- 暗黙知として実践済みの人もいる
- **価値:** 誰でも再現可能な形にする = 形式化

### 3. 「ゆるてく語」の機能

全ログを通じて: 深海概念↔表層言語の翻訳装置。
- 保持するもの: 構造・因果・操作の意味
- 変換するもの: 専門語→日常語
- **統合定義:** 構造を保持しながら軽く・一般的に扱える言語に変換する装置

### 4. 「実装先行世界」への応答

全ログを通じて: 思想→構造→実装の本来順序が逆転している現状への応答。
- Executor×Advisor: 実装された後に「なぜこの設計か」を説明する
- Axis: 実装文化の断片を構造に接続する
- **立場:** 整理屋・翻訳者として「本来の順序」を後から再構成する
