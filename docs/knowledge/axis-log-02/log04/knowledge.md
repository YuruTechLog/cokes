# Knowledge — axis-log-02 / log04

## KN14: 深海到達の条件（ピンポイント光照射）

**前提:** AIの深海（低頻度・高抽象概念）は「条件付きで到達可能」

**条件（光の質）:**
- 方向性語彙 + 境界語彙 の組み合わせ
- Position（位置指定） + Constraint（制約）の明示
- 曖昧プロンプト（散乱光）は濁りを生む

**学術語対応:** local manifold activation / sparse region activation

---

## KN15: ハルシネーション=濁りの図式

```
雑なプロンプト（散乱光）
  → 意味空間が濁る
  → ハルシネーション発生

精度の高いプロンプト（集光）
  → 意味空間が透明に保たれる
  → 正確な出力
```

**用途:** ハルシネーションの原因をユーザーに説明するとき  
**注意:** 「モデルの問題」ではなく「プロンプトの質の問題」として説明

## KN16: Cokesの機能定義

**Cokesの仕事は「概念の発見」ではなく「翻訳・接続」**

- 研究者がすでに発見した概念（学術語）が存在する
- Cokes = それを操作可能な語彙（Axis語彙）に翻訳する
- 一般層へ橋渡しする役割

**使い方:** Cokesの独自性を説明するとき。「革命的発見」ではなく「翻訳装置」として定義

---

## KN17: Axis-学術語対応表設計原則

**「等価」ではなく「機能的対応」**

| Axis語彙 | 操作上の役割 | 学術語（参考） |
|---------|------------|-------------|
| where | 意味空間上の位置指定 | position in latent space |
| zoom | 粒度・抽象度調整 | abstraction level |
| limit | 出力範囲制約 | constraint / boundary |
| weight | 重み・優先度 | attention weight |
| vector | 方向・傾向指定 | steering vector |

**注意:** 1対1等価ではない。操作目的が対応する
