## KN01: 言語化支援の設計根拠

**概念:** Cokesが「発見」ではなく「言語化支援」である根拠

**前提:** 知識は既に人間の中にある  
**問題:** 言語化する人がいない（暗黙知のまま閉じている）  
**Cokesの役割:** 閉じた暗黙知を言語化・形式化・共有可能にする

**レイヤー変換:**
```
暗黙知（体験層）→ 形式知（言語化層）→ 共有知（伝達層）
         ↑Cokesが介在
```

**由来:** 「言語化する人がいないだけ」確認（log11）

## KN03: 外部評価と内部改善の分離

**概念:** フレームワーク採用前後で評価軸が変わる

**分離:**
- 外部評価（信念なし）: 弱点特定・批判的評価 → 取り込んで改善
- 内部改善（信念あり）: 精緻化・実用化 → 体系の深化

**NG:** 外部批判のみで設計を壊す。内部改善のみで弱点を無視する  
**由来:** 信念の非対称性議論（log11）
