# Knowledge — axis-log-03 / log10

## KN01: Cokes×Approach Axisの補完関係

**概念:** CokesとAxisは統合ではなく独立補完関係

**定義:**
- Cokes = 知識整理工学（人間の暗黙知を整理・形式化する体系）
- Approach Axis = 人間軸レンジ × AIベクトル場の論理和を操作化する体系

**補完の流れ:** Cokesが整理した知識 → Axisが操作可能な状態に変換  
**NG設計:** CokesにAxisを吸収させる（各体系の純度が下がる）  
**由来:** 統合試み→分離判断（log10）

## KN02: Handoff = 分岐点の定義（確定）

**概念:** Handoffは知識移転の「終点」ではなく「分岐点」

**定義:** 知識が移転された瞬間に、次の処理（循環の次フェーズ）が始まる起点  
**NG定義:** Handoff = 終わり（完了）

**図での配置:** Cokesの中心概念ではなく、特定状況で発生する独立した概念として別図に  
**由来:** 「HandoffがCokes図中心に入ると不自然」指摘→再定義（log10）

## KN03: 循環構造の設計原則

**概念:** Cokesの作業サイクルは1周で完結しない

**構造:**
```
潜る（深掘り）
  → 切り分け（分析・構造化）
  → また潜る（次レイヤー）
  → 前提変化（発見）
  → 戻る（再定義）
  → 次の潜り…
```

**含意:** 繰り返しによって精度が上がる。「完了」ではなく「次フェーズ開始」として捉える  
**由来:** 循環構造初言語化（log10）
