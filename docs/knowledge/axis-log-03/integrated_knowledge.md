
## IK02: Cokesの定義と領域

**由来:** log02・log03（確立）、log07（境界明示）

### 定義
- Cokes = 知識整理工学（Knowledge Organization Engineering）
- 目的 = 人間の暗黙知を形式化・共有可能にする
- 手段 = 判断プロセスの構造化・記録・再利用

### AIの位置づけ
- AIは「契機」（触媒）であり、思想の主体は人間
- AIは言語層のみ。暗黙知層+言語化層 = 人間が持つ

### 境界（Cokesが扱わないもの）
- AI全般論（AIがどう動くか）
- ツール論（何を使うか）
- 思想進化史（なぜこの思想に至ったか）

## IK09: 骨格→AI統合ルート

**由来:** log13（確立）

### 設計
```
1. 人間が骨格を抽出
   （必要十分条件・判断基準・転換点・疑問点）
        ↓
2. AIが骨格に肉付け
   （統合・言語化・構造化・整形）
        ↓
3. 人間が確認
   （品質チェック・過剰品質除去・正確性検証）
```

### 問題の根拠（AI要約の構造的欠陥）
- AIは言語パターンを圧縮する
- 非言語の構造（思考の流れ・判断の変遷・転換点）は捨象される
- 結果: 外殻のみ残る（骨格欠落）

### Cokes蒸留パイプラインとの対応
- Phase1（log_topics）= 骨格抽出
- Phase2（distillation）= 骨格の構造化
- Phase3（summary）= 意思決定の流れ保持
- Phase4（knowledge）= 再利用可能に整形
- Cross-log Integration = 全体統合
