# Knowledge — axis-log-03 / log12

## KN01: Fable/Axis/Cokesの三者関係（確定）

**概念:** 三つのフレームワークの役割と関係

```
Cokes（上位体系）
  ├─ Axis（操作体系）─── AI側の未知を扱う
  │    └─ 具体実践 → Fable本家
  └─ Fable（Tips集）─── 人間の未知を扱う
```

**定義:**
- Cokes = 知識整理工学（全体統括）
- Axis = 人間軸レンジ×AIベクトル場の操作体系（AI側の欠落を補正）
- Fable = 「自分が何を知らないか確認する」Tips集（人間の未知を明示化）

**未知の区別:**
- Fable: 人間が知らないこと（人間の未知）
- Axis: AIが知らないこと（AI側の暗黙知欠落・地形崩壊）

**由来:** 三者関係確立議論（log12）

## KN02: 構造飽和点の判断基準

**概念:** 概念拡張を止めて収束・統合フェーズに移行するタイミング

**判断基準:**
- 新概念を追加しても既存概念の説明力を超えない
- 追加コスト（理解コスト・統合コスト）> 追加価値
- 「もうこれ以上広げても意味ない」という感覚

**行動:** 飽和点検知後 → 収束・統合フェーズへ移行。新概念投入禁止  
**由来:** 構造飽和点認識（log12）

## KN03: AI自身によるフレームワーク誤認のリスク

**概念:** AIはCokes/Axis/Handoffを説明するとき誤認を起こす可能性がある

**実例:** 「Axisの Planフェーズ」（Axisにフェーズ構造は存在しない）  
**原因:** AI自身がAxisの地形を誤認（Axisを地形で変えられるはずが、自分がAxisを誤認）

**対策:**
- AIが出力したCokes/Axis説明は人間が必ず検証する
- 「Axisにはフェーズ構造がない」等の固定事実を明示してから使用

**由来:** ハルシネーション記録（log12）
