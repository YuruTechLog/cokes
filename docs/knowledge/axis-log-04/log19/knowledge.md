# Knowledge — axis-log-04 / log19

## KN01: Ollama vs SD VRAMアーキテクチャ差異

**概念:** LLM推論と画像生成のVRAM消費パターンの本質的差異

**差異:**
- **Ollama（LLM）**: コンテキスト長×トークン数のKV-cacheを常時保持 → 会話が長くなるほど消費増加
- **Stable Diffusion**: 各ステップで前ステップ上書き → 常に「今のステップ」分だけ使用（上限一定）

**実用:** 8GB GPU: SDは余裕 / Ollamaは短い会話しか無理
**適用:** GPU選定・環境構築記事の技術説明として使用可能

## KN02: 推論×画像生成の最強リソース配分

**概念:** AI活用環境設計の推奨パターン

**配分:**
- 推論（長文・対話・分析）: Claude Pro（クラウド）
- 画像生成（バナー・素材）: RTX5040 + SD（ローカル）

**結論:** ローカルGPUを推論に使う必要なし
**適用:** AI活用環境設計の推奨パターンとして提示可能

## KN03: DALL-E vs ローカルSD 意思決定ガイド

**概念:** バナー・画像生成ツール選定の根拠

**DALL-Eの問題:**
- プロンプト改変（OpenAIが勝手に修正）
- 再現性ゼロ
- ブラックボックス

**ローカルの強み:**
- 構図完全固定（ControlNet）
- 画風後付け可能（LoRA）
- 再現性あり（シード固定）
- 3レイヤー設計が適用できる

**適用:** DALL-E vs ローカルSDの比較記事・意思決定ガイドとして使用可能
