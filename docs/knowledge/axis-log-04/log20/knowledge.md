## KN01: Python+Playwright バナー工場の強度

**概念:** HTMLベースのバナー生成が最強である根拠

**強度:**
- 再現性: 100回実行 = 100回同じ出力（CSSで制御）
- メンテナンス性: CSS1行変更で全体デザイン変更
- 速度・コスト: 1秒未満・GPU不要・クラウドAPI不要

**対比:** AIバナー生成の弱点（非決定性・文字化け・高コスト）を全て回避
**適用:** バナー生成ツール選定の根拠として使用可能。既存のHTML工場の継続根拠

## KN02: ハイブリッドの正しい分業

**概念:** AI（SD）とコード（Playwright）の最適分業パターン

**分業:**
- AI（Stable Diffusion）: 背景テクスチャ・素材画像 → 多様性・センスが活きる
- コード（Playwright）: テキスト配置・レイアウト・最終合成 → 確実性・再現性が必要

**原則:** 「決定権はコードに」「素材だけAIに」
**適用:** AI活用論・ツール選定記事のフレームワークとして使用可能

## KN03: 「指揮する人」モデル

**概念:** AI時代のエンジニア像

**モデル:**
```
作る人    → 全工程を自力でこなす（高時間コスト）
指揮する人 → 各工程に最適なツールを配置・監督
  推論・分析: Claude Pro
  画像生成:   SD（必要な時のみ）
  バナー合成: Playwright
```

**核心:** 「AIを使うこと」が目的でなく「結果を出すこと」が目的
**適用:** AI時代のエンジニア像として記事化可能。どんなAI活用論にも通底するメッセージ
