# Knowledge — axis-log-04 / log15

## KN01: 一般論NOTの根拠

**概念:** AXISと一般的プロンプトTipsの原理的差異

**差異:**
- 業界一般論: 「良いプロンプトは詳しく書く」「トーン指定が大事」
- AXIS: 「構造語彙を先に置く」「軸を設計する」

**根拠:** 一般論は「何を書くか（形式層）」、AXISは「どの軸から設計するか（意味空間層）」
**なぜ一般化されないか:** 原理を理解していない人が経験則を語っているから

**適用:** AXIS記事で「なぜ既存のプロンプトTipsと違うか」の説明に使用

## KN02: メタプロンプトの限界と超越

**概念:** メタプロンプト（AIにプロンプトを書かせる）へのAXIS適用

**構造:**
- メタプロンプト（軸なし）: ガチャの自動化
- メタプロンプト（軸あり）: AXISで軸設計 → AIが軸に沿ってプロンプト展開

**法則:** 軸設計が先 → メタプロンプトが後（道具）
**適用:** 上級者向けAXIS活用記事の核心

## KN03: パーツ単位発注の戦略

**概念:** AIへの発注スコープを最小化することで効率最大化

**対比:**
- 全体発注: 一箇所ズレで全体やり直し
- パーツ発注: リトライのスコープを最小化

**極限形:** 「ボルト3行だけ発注」
**適用:** 実践的プロンプト設計の即使用可能なテクニックとして記事化可能
