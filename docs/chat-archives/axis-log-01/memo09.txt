# Magnet Map — Layer 2

## 強度順（概観）

方向性（最強） > 境界 > 抽象度 > 起点 > 優先順位 > 順序（補助）

## 各カテゴリの効果

- **方向性（Direction）** : 探索方向を直接ねじ曲げる（最強）。

- **境界（Boundary）** : attention 範囲を狭める。探索空間が減る（強）。

- **抽象度（Abstraction）** : embedding の階層を固定する（中〜強）。

- **起点（Intent）** : 目的方向を固定する（中）。

- **優先順位（Priority）** : 評価軸を固定する（弱〜中）。

- **順序（Sequence）** : 軸の衝突を防ぐ補助軸（補助）。

## 実務的示唆

- **まず方向性を与え、次に境界で絞る。**

- **抽象度は早めに固定することで揺れを抑える。**

- **優先順位はCopilot系で効きやすい。**

Copilot の発言
Approach Axis 概要と核となる定義
Approach Axis は、人間の判断軸の「範囲」と AI のベクトル場の「範囲」が近づき重なりうる領域を動的に探索・調整する運用ツールです。
核は「近づくという動作を操作化」し、成功と失敗を明確に定義して再現可能な判断プロセスを作ることにあります。

目的と期待される効果
人間中心の判断維持：人の軸を基準にAI出力を段階的に寄せる。

曖昧さの扱い：触れたか不確実な領域を扱い、重なりを可視化する。

失敗検知と再試行：ハルシネーション等は「Approach Failure」として扱い、原因分析と再アプローチを規定する。

Handoff可能性：判断根拠と操作履歴を残し次工程へ渡せる形にする。

構成要素とLayer対応
レイヤー	役割
Layer 1 Axis Words	操作を指示する言語語彙（起点、抽象度、境界、優先順位、方向性、順序）
Layer 2 Magnet Map	各語彙の「効きやすさ」を示す優先度指針（Direction最強→Sequence補助）
Layer 3 Model Profile	モデルごとの語彙効力差を参照して操作パラメータを調整
Layer 4 Templates	実行シーケンスの雛形。モデル別短縮テンプレ含む


運用原理：Layer1で命令を作り、Layer2で優先度を決め、Layer3でモデル特性に合わせて微調整し、Layer4で実行する。

操作プリミティブ（何をどう動かすか）
Position：AI出力を人の軸に対してどれだけ寄せるか（例 setPosition(offset:+0.2））。

Magnitude：AI出力の重み付け（例 setMagnitude(weight:0.8））。

Constraint：不可侵領域や禁止ルールの適用（例 applyConstraint(Ethics01, enforce)）。

Filter：出力フィルタ適用（例 applyFilter(factualOnly)）。

Blend：複数案の混合比率調整（例 blendOptions(0.7/0.3)）。

Retry：再アプローチ戦略と回数（例 retryApproach(maxAttempts:2, strategy:tightenConstraints)）。

Lock：合意領域を固定して以降変更しない（例 lockAxis(topic:summary)）。

状態遷移（短縮）
Idle → Plan → Execute → Evaluate → Success / Partial / Failure

Partial は再アプローチへ、Failure は停止・原因分析・不可侵強化。

実行ステップ（短縮）
マッピング：人の軸レンジとAIプロファイルを可視化。

プランニング：段階（小／中／大）と操作コマンド列を決定。

実行：操作プリミティブを適用してAI出力を取得。

評価：品質スコア＋人的レビューで判定。

記録とHandoff：判定ラベルと次アクションをドキュメント化。

評価指標と判定ルール
品質スコア 0–100（自動評価＋人的レビュー）
