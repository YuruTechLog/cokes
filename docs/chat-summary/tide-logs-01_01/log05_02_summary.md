# テーマ2 気象データ利用と法的整理

## ログで確認できた内容

### 発端

`data.json` の元データとして公開気象情報を利用する場合、
法的・ライセンス上の問題がないか確認した。

### 調査内容

Copilotより以下の整理が提示された。

- 公開気象データを加工して利用する構成は一般的に成立する
- 元データの利用規約（出典表示など）は守る必要がある
- 加工後の `data.json` は利用条件を明示した方がよい
- バックエンド非公開・フロントOSS構成との組み合わせも問題ない

続けて、

- 直近5日分を日々更新する運用
- Cloudflare経由で公開する構成

についても確認を実施。

さらに、

「予報業務」に該当するかどうかを確認した。

### 判断

提供するのは

- 未来の天気予報ではなく
- 独自計算による指標

であり、

利用時には免責を表示する方針とした。

## 結論

現時点では、

- 公開気象データを加工した指標を提供する
- 予報ではなく指標として扱う
- 利用者へ免責を明記する

という運用方針で整理された。
