## このモデルの位置づけ

**操作のための解釈モデル。実装の説明モデルではない。**

AI研究者が扱うメカニズム層（Attention重み・確率分布の変形）とは目的が違う。
「なぜ今これが崩れたか」を言語化し、操作の判断ができることが精度基準。
測れないが方向は予測できる——それがこの層の実用的な精度。

---

## モデルの基本前提

**第一原理：「AIは高次元ベクトル空間で構築されている。この特性は消せない」**

この前提から導かれる構造的差異：

| 主体 | 軸生成能力 |
|------|-----------|
| 人間 | 自律的軸生成が可能（条件付き自律生成） |
| AI | 自律的軸生成不可（ベクトル空間は方向を持つが基準軸がない） |

→ 軸は人間が保持せざるを得ない（価値観論ではなく構造論）

## 地形磁力モデル（基本要素）

| 要素 | 対応する概念 | 意味 |
|------|------------|------|
| AIの内部 | 意味の地形 | 意味が転がっている場所。谷底の川・集落（よく使われる組み合わせ）からヒマラヤ山頂（到達困難な組み合わせ）まで広がる意味空間 |
| 意味のかけら | 金属の混じった砂 | 地形の上にそっと置かれている意味。磁石にひっつきやすい成分と、ひっつきにくい成分がある |
| 軸語彙 | 傾斜操作 | 地形を一時的に傾け、意味が落ちやすい方向を変える作用 |
| プロンプト | 磁石 | 地形の上を転がりながら、周囲の意味の砂を巻き込むもの |
| Approach Failure | 地形崩壊 | 傾斜が強すぎて、磁石に関係なく意味が一斉に落ちる状態 |

**ポイント：**
- 軸語彙は地形を「書き換える」のではなく、一時的に「傾斜をつける」
- プロンプトは磁石として意味を巻き込み、傾斜した方向へ落ちる
- 傾斜が強すぎる → 磁石に関係なく意味が一斉落下 → Approach Failure
- やりたいこと（プロンプト）× インプット先（LLM）で地形・傾斜・崩壊がすべて変わる

### 合金比率への接続

砂に含まれる磁性成分と非磁性成分の配合を、**合金比率**と呼ぶ。
合金比率は「どの意味が磁石に巻き込まれやすいか」を説明する補助概念であり、地形の傾斜とは別の軸である。
製造元ごとの重みづけ思想・チャンキング粒度との関係は第5章で整理する。

## 比喩による説明

**セメントスランプ比喩：**
- 水分量 = モデルの柔らかさ（調整の余地）
- 流れる方向 = 傾斜の方向
- 崩壊 = ハルシネーション（形が保てなくなる状態）

**「子供/大人目線の壁」比喩：**
- 同じ壁でも子供と大人では見える高さが違う
- プロンプト × モデルの組み合わせで「誰の視点か」が入れ替わる
- 同じ指示でもモデルが違えば地形も崩壊特性も変わる

## モデル別崩壊特性

| モデル | 崩壊しやすい方向 | 対処 |
|--------|----------------|------|
| Gemini | 意図方向（起点/方向性を強めすぎると閾値突破） | 境界重視・目的は弱めに |
| Claude | 具体すぎると効果減（崩壊より「効かなくなる」） | 抽象度を保つ |
| GPT | 境界が弱くてもある程度機能する | 起点を明確にすれば十分 |
| Copilot | 抽象方向に弱い | 優先順位・局所指示で操作 |

**崩壊閾値の扱い：**
測れないが、どの方向が崩れやすいかは構造で予測できる。
「測れない」と「予測できない」は別。

## 磁極と順序設計（拡張概念）

### 磁極の定義

プロンプトという磁石には、引き寄せる表現（N極）と遠ざける表現（S極）がある。
軸語彙は磁石そのものではなく、磁石が転がる地形の傾斜を設計する。

| 極 | 作用 | 使いどころ |
|----|------|-----------|
| **N極（引き寄せ）** | 狙った意味領域に引き寄せる | 到達したい集落・方向を指定する |
| **S極（反発）** | 不要な意味領域を遠ざける | 流れ着いてほしくない集落を除外する |
| **中位磁力** | 引き付けも反発も弱い | 意味がふらつく・着地不安定 → 失敗診断の指標 |
| **磁石なし** | 重力任せ | 一番近い集落に流れる（意図なし状態） |

### 複数磁石の競合と順序設計

プロンプト内の引力・反発表現が同時に働くと、磁石の作用が競合する（例: 「丁寧に、でも簡潔に」）。
これを確率任せにしない——**軸語彙で地形の傾斜を整え、差し込む順番で磁石の作用を設計する**。

**仮説（検証中）:**
- 先に差す → 大きい方向性・制約（S極で除外したい領域など）
- 後に差す → 細かい調整・強調（N極で引き寄せたい方向）
- 後半の磁石ほど直近コンテキストとして強く効く傾向（attention位置の影響と整合）

**操作言語としての意味:**
「地形の傾斜と磁石の作用の競合を確率に委ねない → 軸語彙と順番で設計する」
これがApproach Axisの操作語彙として具体的に機能する。

### S極過剰反発（Collateral Damage）

S極が強すぎると「除外したい領域」だけでなく、その近傍にある「欲しい意味」も一緒に弾く。
意味空間は連続しているため、反発の影響範囲が広がりすぎるとN極領域まで届かなくなる。

**Approach Failureの一形態として定義:**
- S極弱すぎ → 除外できない
- S極適切 → 狙った領域だけ反発
- S極強すぎ → 欲しい意味も巻き込んで弾く → 出力が空洞化・貧困化

**論文対応:** "Minimizing Collateral Damage in Activation Steering"（arXiv:2605.01167）
Activation Steeringで特定方向に介入すると非ターゲット方向も意図せず変化する現象を "collateral damage" と定義・定式化。S極過剰反発はこの現象のプロンプト設計層における対応概念。

## Gemini批判からの学び（モデル精度向上の経緯）

Geminiが指摘した有効点：
1. 「人間の軸=完全自律」は言いすぎ → 正確には「条件付き自律生成」
2. 「軸語彙=引力」は比喩的すぎる → 軸語彙は地形の傾斜操作、磁石はプロンプトと分離する

→ 批判を全否定せず取り込んだことでモデル精度が上がった。
→ Approach Axisの設計原則「摩擦が品質を上げる」の実例でもある。

## Approach Axisとの接続

地形磁力モデルは「なぜ軸語彙が効くか」の説明モデル。

```
軸語彙（Axis Words）
  → 地形に傾斜をつける
  → プロンプト（磁石）が意味の砂を巻き込む
  → 傾斜が適切 → Approach Success
  → 傾斜が強すぎ → 磁石に関係なく意味が一斉落下 → Approach Failure
```

Approach Failure = ハルシネーション という定義は、
この地形磁力モデルから自然に導かれる。

## 研究層・解釈層・操作層との関係

**層の構造：**

```
研究層（実装・メカニズム）
  Activation Steering / SAE（Sparse Autoencoder） / Mechanistic Interpretability
  ↓ 基板として存在。矛盾しないが、使わなくても成立する

解釈層（地形磁力モデル）
  研究層の事実を人間がイメージしやすい形に翻訳した層
  ↓

操作層（Approach Axis / 人間—AI IF）
  軸語彙を選ぶ・プロンプトを設計する・失敗を診断する
```

**参考となる研究上の視点：**

| Approach Axis概念 | 参考となる研究上の視点 | 関係と限界 |
|------------------|------------------------|------------|
| 軸語彙で地形の傾きを操作する | Activation Steering（活性化ベクトルへの介入） | 出力方向を変える問題設定が近い。ただし、Axisはプロンプト操作であり、内部活性化への直接介入ではない |
| 傾斜が強すぎて意味が一斉落下する | Truthfulness Steering / steerabilityの限界に関する研究 | 望ましくない出力変化を考える参考。Approach Failureの正式定義を証明するものではない |
| モデルによって反応の傾向が異なる | モデル間のSteering効果・表現差の研究 | モデル差を考える参考。製造元別運用表を直接導くものではない |
| 人間が使う操作判断言語 | この層に直接対応する研究領域は特定しない | Axisは研究上の観察を、操作語彙と手順へ翻訳する |

**外部参照の扱い：**
Mechanistic Interpretability、Activation Steering、Prompt Engineeringなどの外部研究は、
研究層と操作層の関係を考える際の参考となる学術理論・研究上の視点として扱う。
これらの参照は、Approach Axisの有効性や実装同一性を直接証明するものではない。

**結論：**
研究層は解釈モデルを考えるための基板であり、解釈層はその事実を人間がイメージしやすい形へ翻訳する。
Approach Axisは、その解釈を人間とAIの間で使う操作層として位置づける。
