# 第2章: なぜ効くのか — 地形磁力モデル

**目的:** Approach Axisが機能する理由を「操作のための解釈モデル」として示す。実装の説明モデルではないことを明示する。

## 骨子

- **モデルの位置づけと第一原理の掲載**
  AIの内部機構を直接説明するのではなく、出力の偏りを操作するための解釈モデルであることを最初に明示する。
  第一原理「AIは高次元ベクトル空間で構築されている。この特性は消せない」は、地形の比喩に入る前提として短く掲載する。
  （参照: `drafts/terrain-magnetic-model.md` §このモデルの位置づけ §モデルの基本前提）

- **地形・意味・軸語彙の3要素表を掲載**
  地形（意味が転がる場所）/ 意味の砂（地形の上に置かれた意味）/ 軸語彙（地形の傾きを操作する語彙）の対応を表にする。
  砂には磁石にひっつきやすい成分とひっつきにくい成分があり、その合金比率は第5章で整理する。
  プロンプトは磁石として意味を巻き込み、傾斜した方へ落ちていく。
  ポイント: 軸語彙は地形を書き換えるのではなく、一時的に傾きを操作する。
  （参照: `drafts/ch02_draft_v2.md` §2.1〜§2.3）

- **Approach Failureの比喩を掲載**
  傾斜が弱い場合は磁石が近い谷へ流され、適切なら狙った意味を巻き込み、強すぎる場合は磁石に関係なく意味が一斉落下する。
  これをこの章でのApproach Failureのイメージとし、正式な判定・診断は第8章へ送る。

- **関連研究の位置づけを掲載**
  EBMやActivation Steeringを、Axisの直接的な証明ではなく、解釈モデルを考えるための関連概念として短く示す。
  詳しい研究層との対応は第3章へ送る。

- **比喩体系の扱い**
  第2章では地形・砂・磁石の比喩だけを正本として使う。
  食紅・電波フィルター等の補助比喩は、第10章の実践Tipsに集約する。

> **[方針確定]** 第2章は地形・砂・磁石の基本比喩に限定し、補助比喩の一覧と実践的な使い分けは第10章へ置く。
