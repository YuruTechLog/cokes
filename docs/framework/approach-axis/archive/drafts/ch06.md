# 第6章: 操作プリミティブ7種（骨子）

> 本文ドラフト: `ch06_draft_v1.md`
> 用語の正本: `../glossary.md`

**目的:** Approach Axisの操作命令の基本単位（プリミティブ）を一覧化し、各プリミティブの使い方を示す。

## 骨子

- **7種の表を掲載**
  Position / Magnitude / Constraint / Filter / Blend / Retry / Lock の7種。
  各プリミティブの操作内容をまとめた表（コマンド例はそのまま掲載。下記[人間判断]参照）。
  （参照: `axis-log-01/approach-axis-framework.md` §操作プリミティブ（7種））

- **各プリミティブの役割説明**
  Position: AI出力を人の軸に対してどこへ寄せるか（移動先）
  Magnitude: 寄せ方の操作ニュアンスを自然言語ラベルで指定する（slight / moderate / strong）
  Constraint: 不可侵領域・禁止ルールの適用
  Filter: 出力フィルタ適用
  Blend: 複数案の混合比率調整
  Retry: 再アプローチ戦略と回数
  Lock: 合意領域を固定して以降変更しない
  （参照: `axis-log-01/approach-axis-framework.md` §操作プリミティブ）

- **コマンド例の位置づけの明記**
  表中のコマンド例（`setPosition(target: conceptual_explanation)` 等）は擬似コードとして掲載。
  （参照: `axis-log-01/approach-axis-framework.md` §操作プリミティブ）

> 素材からは判断不能のため仮置き: 「表記は擬似コード。実装時はプロジェクトごとに定義」と注記。

- **機構非依存の注意**
  プリミティブはAI内部の関数やAPIではなく、人間が変更内容を記録するための運用上の操作単位として扱う。

- **ニュアンス語と代表指示の対応**
  `slight / moderate / strong` を数値ではなく自然言語の操作ニュアンスとして定義し、それぞれに代表的な指示例を添える。

- **ニュアンス語の受入条件**
  各ラベルは内部状態の数値ではなく、出力を人間が確認するための観測基準として扱う。主題・境界・前提をどこまで維持するかを、タスク固有の受入条件とともに記録する。

- **操作順序の2パターン**
  「制約先行型（絞ってから作業する）」と「探索先行型（可能性を残してから絞る）」を使い分ける。安全境界は先に置き、創作や仮説検討では候補生成後に表現上の制約を適用できる。

- **Retryの停止条件**
  指定方向への接近、Constraint違反の有無、変化の有無、前提変更の有無を停止・別軸見直し・新しいHandoffへの移行条件として記録する。
