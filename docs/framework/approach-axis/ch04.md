# 第4章: Approach Axis運用アーキテクチャ（4層）

> 本文 v1 / 2026-07-23（レビュー済み・昇格済み）
> 用語の正本: `glossary.md`

Approach Axisは、研究層を直接実装するものではない。
第3章で整理した解釈層を、人間とAIの間で使える操作層にしたものである。

その操作層は、4つの層に分かれている。

```text
第1層  軸語彙層       ── 何を伝えるか
第2層  意味地形図層   ── 意味がどこに配置されているか
第3層  AI特性層       ── 相手のAIに合わせてどう調整するか
第4層  実行雛形層     ── どう実行・評価・引き渡すか
```

4層は別々の機能一覧ではない。人間の判断軸をAIへ渡し、出力を受け取り、次の判断へつなぐ一つの流れである。

---

## 4.1 4層の全体像

| 層 | 日本語名 | 既存名 | 役割 | 主な成果物 |
|---|---|---|---|---|
| 第1層 | 軸語彙層 | Axis Words | 人間の意図・判断軸を、AIに渡せる操作語彙にする | Intent / Abstraction / Boundary / Priority / Direction |
| 第2層 | 意味地形図層 | Semantic Terrain Map | 意味の配置・高低・近接関係を把握する | 意味の配置、谷・山、近接関係 |
| 第3層 | AI特性層 | AI Profile / AIプロファイル | 使用するAIの反応傾向に合わせて操作条件を調整する | モデル特性、合金比率、磁石サイズ、安全装置傾向 |
| 第4層 | 実行雛形層 | Templates | 設計した操作を再利用可能な手順にする | 運用カード、評価欄、Handoffテンプレート |

第1層から第4層へ進むほど、抽象的な判断軸から具体的な運用へ近づく。
ただし、後ろの層が前の層を置き換えるわけではない。AI特性に合わせても、人間が保持する軸そのものは変わらない。

---

## 4.2 第1層：軸語彙層（Axis Words）

軸語彙層は、人間の判断軸をAIへの指示に使える語彙へ変換する層である。

軸語彙は、意味を直接引き寄せる磁石ではない。意味の地形に一時的な傾きをつけ、プロンプトという磁石がどの方向へ落ちやすいかを操作する。

基本の分類は次の5つである。

| 分類 | 役割 |
|---|---|
| Intent | 何を達成したいかを決める |
| Abstraction | どの粒度・抽象度で扱うかを決める |
| Boundary | どこまでを許容し、どこからを不可侵領域とするかを決める |
| Priority | 複数の条件のどれを優先するかを決める |
| Direction | 出力をどの方向へ寄せるかを決める |

Sequenceは、複数の軸語彙を扱う順番を補助的に設計する。
第1層の成果物は、単なるキーワード一覧ではなく、今回の操作で使う軸語彙の組み合わせである。

---

## 4.3 第2層：意味地形図層（Semantic Terrain Map）

意味地形図層は、意味がどこに配置され、どこが高く・低く、どの意味同士が近いかを把握する層である。

ベクトルDBのように、意味の配置を参照するイメージである。ただし、LLMに外部ベクトルDBが必ず存在するという意味ではなく、内部表現を直接観測できるという意味でもない。

地形の高低は、意味が落ちやすいか落ちにくいか、谷に近いか遠いかという相対的な配置として扱う。
この層は地形を操作する層ではない。軸語彙とプロンプトが傾斜や経路を操作する対象を、人間がイメージできる形にする層である。

基本の見方は次のとおりである。

```text
意味の配置を読む
  → 近い谷・遠い谷を把握する
  → 高低と近接関係を把握する
  → 軸語彙で傾ける方向を決める
  → プロンプトという磁石を転がす
```

LLMごとのチューニングによる地形の違いは、第3層のAI特性層で扱う。

---

## 4.4 第3層：AI特性層（AI Profile / AIプロファイル）

AI特性層は、同じ軸語彙でも使用するAIによって反応が変わることを扱う層である。

第2章の地形磁力モデルで言えば、AIごとに地形の性質や砂の合金比率が異なる。磁石（プロンプト）を同じように転がしても、巻き込まれる意味の量や種類、傾斜への反応は同じとは限らない。

AI特性層で確認する項目は次のとおりである。

- **合金比率**：磁石にひっつきやすい意味と、ひっつきにくい意味の配合
- **磁石サイズ**：一つの指示や意味の塊をどの粒度で扱うか
- **重みづけ思想**：helpfulness、harmlessness、factualityなど、出力で優先されやすい方向
- **安全装置チューニング**：拒否・フィルタリングの強度と方針

AI特性は、AIに人格や意図があるという意味ではない。設計者が、どの条件で出力の傾向が変わるかを事前に把握するための参照情報である。

---

## 4.5 第4層：実行雛形層（Templates）

実行雛形層は、第1〜3層で設計した内容を、実際に使える手順へ落とす層である。

最低限、次の情報を残す。

```text
目的：
人間の軸レンジ：
AIプロファイル：
使用する軸語彙：
優先順位・順序：
実行内容：
評価：Success / Partial Success / Approach Failure
次アクション：
Handoff：
```

実行雛形は、プロンプト本文だけを保存するものではない。どの軸を持ち、どのAI特性を前提にし、どの判定を通過したかを残すことで、次の人間や次の工程が再利用できる状態にする。

---

## 4.6 4層を一つの流れとして使う

4層の運用は、次の流れになる。

1. **軸語彙層**で、人間の意図・判断軸を言葉にする
2. **意味地形図層**で、対象となる意味の配置・高低・近接関係を把握する
3. **AI特性層**で、使用するAIに合わせて条件を調整する
4. **実行雛形層**で、プロンプトを実行し、出力を評価し、Handoffを記録する

評価結果がPartial SuccessやApproach Failureなら、第1層へ戻って軸語彙を見直す場合もある。4層は一方向に一度だけ通過する工程ではなく、出力を見ながら再設計する循環である。

```text
軸語彙
  → 優先度・順序
  → AI特性への調整
  → 実行・評価・Handoff
  → 必要なら軸語彙へ戻る
```

---

## 4.7 Axisの三層構造との違い

4層構造と、Axisの三層構造は別の切り口である。

| 切り口 | 層 | 何を説明するか |
|---|---|---|
| 研究・解釈・操作の3層 | 研究層 / 解釈層 / 操作層 | 研究上の事実が人間の操作へ変換される流れ |
| Axisの三層構造 | 操作層 / 説明層 / 思想層 | Axisが何を使い、何で説明し、何を前提にするか |
| Approach Axis運用アーキテクチャ | 軸語彙層 / 意味地形図層 / AI特性層 / 実行雛形層 | 操作層としてのAxisをどう運用するか |

同じ「層」という言葉を使うが、分類の目的が違う。本文では必ず名称を付けて区別する。

---

## 4.8 まとめ

- Approach Axis運用アーキテクチャは、4つの層で操作を組み立てる
- 軸語彙層は、何を伝えるかを決める
- 意味地形図層は、意味の配置・高低・近接関係を把握する
- AI特性層は、相手のAIに合わせて条件を調整する
- 実行雛形層は、実行・評価・Handoffを再利用可能にする
- 4層は独立した一覧ではなく、設計・実行・評価を循環させる運用の流れである

> 第5章では、軸語彙層で使う5軸と、AI特性層で参照する合金比率・磁石サイズを詳しく整理する。

## 用語の扱い（補足）

Axisの三層構造は、用語として固定する名称ではなく、操作層・説明層・思想層をまとめて説明する分類枠として扱う。
各層の意味は `glossary.md` に補足定義を置く。

`意味地形図層`、`AI特性層`、`実行雛形層` は、既存の概念語を4層構造の中で呼ぶための表示ラベルとして扱い、用語集には追加しない。
