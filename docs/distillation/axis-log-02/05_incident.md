# log05 Incident — axis-log-02

## I01: フォルダ構造と概念構造の混同リスク

- 発生: tools/axis-framework/ 配下に海モデルを置く案検討
- 問題: フォルダ構造が「物理整理」になり、概念の三層（操作/説明/思想）が見えにくくなる
- 結果: フォルダ名だけでなくREADMEで概念層を明示する設計が必要
- 教訓: ファイル構造は概念構造の写像。物理整理と概念整理を混同しない

## I02: 公開順序の設計ミスリスク

- 発生: 「AIは海だった」を先に公開しようとした時
- 問題: 前提となる軸語彙・磁石記事が先に出ていないと文脈が飛ぶ
- 結果: 先行記事（磁石/軸語彙）→海モデルの順序で公開
- 教訓: シリーズ記事は「読者の理解の積み上げ順」で公開順を設計する
