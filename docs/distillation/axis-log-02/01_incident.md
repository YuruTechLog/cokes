# log01 Incident — axis-log-02

## I01: 固体モデルの論理矛盾

- 発生: 地形モデル（山と谷）採用後
- 問題: 「プロンプトが地形を変える」という直感と「地形は固定構造」という設計の矛盾
- 結果: 地形モデル廃棄 → 流体モデルへ移行
- 教訓: 比喩は「誰が操作するか」の主体設定を先に固めてから選ぶ

## I02: 自転車モデルの主体問題

- 発生: 流体の前段として自転車モデル提案
- 問題: 自転車に乗るのがAIになってしまう → 人間主体が消える
- 結果: 即棄却
- 教訓: 比喩の「主体」は常に人間側で設計する

## I03: 片栗粉の二重構造による説明の難化

- 発生: 片栗粉を比喩に使う検討中
- 問題: ダイラタンシー（衝撃反発）と温度粘度変化の2種類のメカニズムが混在
- 結果: 単純比喩としては使いにくい。「2段構造」として後続ログで再検討
- 教訓: 1つの素材に2メカニズムが乗っている場合は分解してから比喩設計
