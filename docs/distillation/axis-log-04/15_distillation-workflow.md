## 抽出した意味構造

### 構造1: 一般論NOTの根拠
```
業界一般論: 「良いプロンプトは詳しく書く」「トーン指定が大事」等
AXIS視点: 「構造語彙を先に置く」「軸を設計する」
→ 原理レベルで異なる。一般論は「何を書くか」、AXISは「どの軸から設計するか」
→ 一般化されていない理由: 原理を理解していない人が経験則を語っているから
```

### 構造2: メタプロンプトの限界と超越
```
メタプロンプト（軸なし）: AIがAIにプロンプトを書かせる → ガチャの自動化
メタプロンプト（軸あり）: AXISで軸設計 → AIがその軸に沿ってプロンプト展開
→ 軸設計が先 → メタプロンプトが後（道具）
```

### 構造3: パーツ単位発注の戦略
```
全体発注: 「このページのUIを全部作って」→ 一箇所ズレで全体やり直し
パーツ発注: 「ヘッダーのナビゲーションだけ」「このボタンの色だけ」
→ リトライのスコープを最小化 → 効率最大化
→ 「ボルト3行だけ発注」という極限形
```
