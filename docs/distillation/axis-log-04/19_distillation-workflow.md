## 抽出した意味構造

### 構造1: Ollama vs SD のVRAM特性
```
Ollama（LLM）:
  コンテキスト長×トークン数 分のKV-cacheを保持
  → 会話が長くなるほどVRAM消費増加（解放されない）
  → 8GB: 短い会話しか無理

Stable Diffusion（画像生成）:
  各ステップで前ステップの中間表現を上書き
  → 常に「今のステップ」分だけVRAM使用（上限一定）
  → 8GB: 余裕で動作
```

### 構造2: リソース配分の最適解
```
推論（長文・対話・分析）: Claude Pro（クラウド）
画像生成（バナー・素材）: RTX5040 + SD（ローカル）
→ それぞれの強みを最大化
→ ローカルGPUを推論に使う必要なし
```

### 構造3: ローカル移行の根拠
```
DALL-E:
  プロンプト改変（OpenAIが勝手に修正）
  再現性ゼロ（同じプロンプトで違う出力）
  ブラックボックス（なぜその画像かわからない）

ローカル（ControlNet+LoRA）:
  構図完全固定（ControlNet）
  画風後付け可能（LoRA）
  再現性あり（シード固定）
  設計可能（3レイヤー設計が適用できる）
```
