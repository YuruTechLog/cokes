## 主要テーマ構造

### Theme A: 完成度 = チャンク粒度の外部指標
**定義**: 完成度 = 「このチャンクだけで意思決定・行動変容が可能か」

**判定基準**:
- 可能 → 適切粒度
- 不可（情報不足） → 統合が必要（粒度大にする）
- 不可（情報過多・判断不明確） → 分割が必要（粒度小にする）

**log06未解決問題への回答**: 粒度外部指標として「完成度」を提案。検証は今後。

---

### Theme B: 発見 vs 言語化（Cokesの役割確定）
- 発見: 新しい事実・法則を見つける（研究・学術）
- 言語化: 既にある暗黙知を明示化・形式化する（Cokes）

**確定**: CokesはRが見つける体系ではなく、「言語化・形式化」が主。「新理論を作らない」Core Ruleと一致。

---

### Theme C: 過剰形式化の副作用
- 形式化を進めすぎると思考の前提が崩れる（「火傷」）
- 形式化にも収穫逓減がある
- 「形式化しすぎない」ラインの判断が必要 → これも人間の暗黙知

---

### Theme D: Advisor表
- Advisorの役割一覧: 判断基準・品質基準・前提定義の保持
- Executorとの分離: 実行責任（Executor）vs 判断責任（Advisor）
- Cokesではこの分離が構造上の前提
