## 主要テーマ構造

### Theme A: 揺れ観察と曖昧性検出
**観察**: 同一プロンプトをSonnet複数回実行 → 応答にばらつき（揺れ）

**揺れの種類**:
1. 確率的サンプリングによる揺れ（温度パラメータ起因）
2. 意味空間の傾斜不足による揺れ（Axis語彙不足）

**実用化**: 同一質問を複数回実行 → 応答分散を測定 → 分散大=入力の曖昧性高 → 曖昧性検出ツールとして使用可能

---

### Theme B: 前提責任の所在
**確定**: 前提の設定・変更・崩壊検知は人間の責任

**理由**:
- AIは外部から与えられた前提を使うのみ
- AIが「前提が変わった」を自律検知してユーザーに通知することは現状困難
- 設計上、前提変化の検知ループに人間を含める必要あり

**運用含意**: Cokesの作業サイクルに「前提確認ステップ」を設計する

---

### Theme C: Cokes領域内/外の明確化
**領域内（Cokesが扱う）**:
- 人間の知識整理体系
- 暗黙知の言語化・形式化
- 必要十分条件の設計

**領域外（Cokesが扱わない）**:
- AIの内部実装最適化
- プロンプトエンジニアリング詳細
- モデルファインチューニング
